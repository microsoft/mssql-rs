# Data Model: Arrow Benchmark Suite

**Feature**: 003-move-arrow-benchmarks

## Entities

This feature moves benchmark code rather than introducing new domain entities. The benchmark data model consists of test data structures and measurement outputs.

### 1. Benchmark RecordBatch (Input Data)

A fixed 5-column Arrow RecordBatch used as input across all benchmark paths. Generated by `build_record_batch(num_rows)`.

| Column | Arrow Type | SQL Server Type | Generation Pattern |
|--------|-----------|----------------|-------------------|
| id | Int32 | INT | Sequential: 0, 1, 2, ... |
| amount | Int64 | BIGINT | `i * 100 + 42` |
| price | Float64 | FLOAT | `99.95 + i * 0.01` |
| name | Utf8 | NVARCHAR(200) | `"product-name-{i:06}"` |
| total | Decimal128(18,2) | DECIMAL(18,2) | `i * 10000 + 12345` |

### 2. Serialization Paths (Strategies)

Each path converts the RecordBatch into TDS wire-format row tokens using a different approach:

| Path | Name | Strategy | Allocation Profile |
|------|------|----------|-------------------|
| A | Pre-serialized | Arrow arrays → per-row `Vec<u8>` of TDS bytes → `write_raw_bytes` | One allocation per row |
| B | Materialized | Arrow arrays → `Vec<ColumnValues>` enum → `write_column_value` | Enum + allocation per field |
| C | Streaming | Arrow arrays → direct unchecked writes to packet buffer | Zero allocation |
| D | ArrowBulkCopy | `ArrowBulkCopy::execute(&RecordBatch)` | Crate-managed |

### 3. Benchmark Result (Output)

| Field | Type | Description |
|-------|------|-------------|
| path_name | string | "direct" / "materialized" / "streaming" / "arrow_bulk_copy" |
| iteration | usize | 1-based iteration number |
| elapsed_ms | f64 | Wall-clock time for bulk copy |
| throughput_krows_s | f64 | `num_rows / elapsed_secs / 1000` |
| is_warm | bool | `iteration > 1` (first iteration excluded from warm averages) |

### 4. Summary Statistics (Per Path)

| Field | Type | Description |
|-------|------|-------------|
| avg_time_ms | f64 | Mean across all iterations |
| avg_throughput | f64 | Krows/s (all iterations) |
| warm_time_ms | f64 | Mean excluding first iteration |
| warm_throughput | f64 | Krows/s (warm only) |
| speedup_vs_materialized | f64 | `materialized_avg / this_avg` |

## State Transitions

None — benchmarks are stateless single-run measurements. Temp tables are created at start, truncated between iterations, and dropped automatically at session end.

## Validation Rules

- `num_rows` must be > 0
- `num_iterations` must be ≥ 2 (to compute warm averages)
- Row count assertions: each temp table must contain exactly `num_rows` after the final iteration
